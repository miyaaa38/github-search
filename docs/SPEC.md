# 機能仕様・制限事項

このドキュメントは「このアプリで何ができて、何ができなくて、どんな挙動をするか」をまとめたユーザー向け仕様書です。

---

## 機能一覧

### 検索ページ（`/`）

| 機能                 | 詳細                                                                                            |
| -------------------- | ----------------------------------------------------------------------------------------------- |
| キーワード検索       | テキスト入力 + 検索ボタン or Enter キーで実行                                                   |
| 空送信バリデーション | 空文字・空白のみの入力では検索を実行せず、フォーム直下にエラーメッセージを表示                  |
| 検索結果一覧         | オーナーアイコン・リポジトリ名をカード形式で表示（30件/ページ）                                 |
| ページネーション     | 最大 34 ページ（GitHub API 上限 1,000 件）。URL の `?page=` パラメータと連動                    |
| URL 状態管理         | 検索キーワード・ページ番号が `?q=&page=` に反映。ブラウザバック・URL 共有で同じ状態を復元できる |
| ローディング表示     | 検索中はスケルトン UI を表示                                                                    |

### 詳細ページ（`/repositories/{owner}/{repo}`）

| 機能               | 詳細                                        |
| ------------------ | ------------------------------------------- |
| リポジトリ名       | `owner/repo` 形式で表示                     |
| オーナーアイコン   | アバター画像を表示                          |
| 説明文             | `description`（null の場合は非表示）        |
| 言語               | 主要言語（null の場合は「指定なし」と表示） |
| Star 数            | 1k / 1M 形式で短縮表示                      |
| Watcher 数         | 詳細 API（`/repos/{owner}/{repo}`）から取得 |
| Fork 数            | 1k / 1M 形式で短縮表示                      |
| Issue 数（Open）   | 1k / 1M 形式で短縮表示                      |
| GitHub リンク      | 外部リンク（新しいタブで開く）              |
| トップへ戻るリンク | 一覧ページへ戻る                            |

> **Note**: Watcher 数は検索 API では `stargazers_count` と同値になることがあるため、詳細 API から取得しています。

## 制限事項・意図的な非対応

| 項目                   | 対応状況 | 理由                                                                                  |
| ---------------------- | -------- | ------------------------------------------------------------------------------------- |
| ソート条件の変更       | 未対応   | GitHub API デフォルト（best-match 順）で十分な要件のため。`sort` パラメータで拡張可能 |
| 入力デバウンス         | 未対応   | ボタン送信型の UI のため不要（タイプのたびにリクエストを出す設計ではない）            |
| 検索履歴の保存         | 未対応   | URL で状態を再現できる設計で代替                                                      |
| 無限スクロール         | 未対応   | ページネーションを採用。URL に状態が乗るため同じページに戻れる利点を優先              |
| ページ 34 超の検索     | 表示なし | GitHub API の検索結果上限が 1,000 件（34 ページ）のためリンクを生成しない             |
| 多重送信               | 防止済み | 検索中はボタンを disabled にして連続送信を抑制                                        |
| プライベートリポジトリ | 未対応   | 公開リポジトリのみ対象                                                                |

## エラー挙動

### 検索ページ

| 状況                        | 表示内容                                                                        |
| --------------------------- | ------------------------------------------------------------------------------- |
| 検索結果 0 件               | 「「{keyword}」に一致するリポジトリは見つかりませんでした」                     |
| 認証失敗（401）             | 「GitHub API の認証に失敗しました。GITHUB_TOKEN の有効性を確認してください。」  |
| API レート制限超過（403）   | 「GitHub API のレート制限に達しました。しばらく待ってから再試行してください。」 |
| クエリ構文エラー（422）     | 「検索クエリが不正です。入力内容を見直してください。」                          |
| その他 API エラー（5xx 等） | 「GitHub API エラーが発生しました（{status}）。」                               |

すべてのエラー表示には GitHub API から返された `message`（例: "Rate limit exceeded for user ID ..."）を **原因** として併記します。再試行ボタンはサーバー側の一時的な失敗時にのみ有効です。

### 詳細ページ

| 状況                          | 表示内容                                                                        |
| ----------------------------- | ------------------------------------------------------------------------------- |
| リポジトリが存在しない（404） | 「リポジトリ {owner}/{repo} が見つかりませんでした。」                          |
| 認証失敗（401）               | 「GitHub API の認証に失敗しました。GITHUB_TOKEN の有効性を確認してください。」  |
| API レート制限超過（403）     | 「GitHub API のレート制限に達しました。しばらく待ってから再試行してください。」 |
| その他 API エラー             | 「GitHub API エラーが発生しました（{status}）。」                               |

### 共通

| 状況                      | 表示内容                                                         |
| ------------------------- | ---------------------------------------------------------------- |
| 存在しない URL にアクセス | 404 ページ（「ページが見つかりません」）+ トップページへのリンク |

## 今後の拡張余地

| 機能                           | 実装方針                                                                            |
| ------------------------------ | ----------------------------------------------------------------------------------- |
| ソート条件（Stars 順・更新順） | GitHub API の `sort` パラメータ（`stars` / `updated`）を `?sort=` として URL に追加 |
| 言語フィルタ                   | GitHub API の `q` パラメータに `language:TypeScript` 形式で付与                     |
| 検索履歴                       | `localStorage` に保存するだけで実装可能                                             |
| お気に入り登録                 | `localStorage` または外部 DB（Supabase 等）で実装可能                               |

## アクセシビリティの設計判断

### ページネーションの無効ボタン

1ページ目の「前へ」・最終ページの「次へ」は `<button disabled>` ではなく `<span aria-disabled="true">` で実装しています。

`disabled` 属性を使うとフォーカスを受け取れなくなるため、キーボード操作中のユーザーが「前へ」ボタンの**存在**を認識できなくなります。このアプリは一覧閲覧がメインのため、前後ページの存在を常に把握できることを優先し、フォーカス可能な `<span>` + `aria-disabled` を選択しました。

> 注意: フォームの送信ボタンなど「操作を明示的にブロックすべき UI」では `<button disabled>` が適切な場面もあります。この選択はページネーションというコンテキストに依存した判断です。
